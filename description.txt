Pour le travail collaboratif, nous avons principalement échangé sur Teams, et Noa a été désignée cheffe de projet.
Nous avons d'abord réparti les tâches, les versions du script et les variables à gérer, puis organisé notre avancement dans Jira pour suivre le projet.
Un dépôt GitHub a été créé pour documenter nos progrès et gérer le code. 
Nous avons utilisé Git pour faire des commits à chaque changement, ce qui nous a permis de suivre l'évolution du script et de voir l'état de chaque fonctionnalité. 
À chaque étape, nous avons relu et testé le code de chacun pour garantir la cohérence du projet avant de fusionner nos contributions.
Bien que nous n'ayons pas fixé de dates précises pour chaque tâche, une bonne organisation nous a permis de tout terminer dans les délais. 
Jira a été très utile pour planifier et s'assurer que rien n'était oublié, et pour savoir qui faisait quoi.
Nous avons également créé un fichier README pour expliquer comment exécuter le programme, comprendre l'objectif du projet, et utiliser cron pour l'automatisation. 
Un fichier supplémentaire contient toutes les descriptions détaillées demandées pour le projet.
Des tests ont été réalisés après chaque tâche pour vérifier le bon fonctionnement. 
Par exemple, nous avons découvert qu'une fonction ne marchait pas sur macOS, ce qui empêchait certains membres d’utiliser le script. 
Grâce aux tests, nous avons pu corriger ce problème rapidement et rendre le script compatible pour tout le monde.
Cette organisation simple mais efficace nous a permis de travailler ensemble facilement, de rester coordonnés, et de finaliser le projet dans les temps.
